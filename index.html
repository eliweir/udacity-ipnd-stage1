Lesson 1 - Basics of the Web and HTML

The Web

The web is a collection of computers that communicate with each other. When someone views a web page like www.udacity.com, their browser sends an HTTP Request (Hyper-Text Transfer Protocol) to a server. The server send the appropriate HTML document to the user's computer and the browser interprets the page and displays it.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML documents contain textual content describing "what you see" and markup describing "what it looks like".

Tags & Elements

HTML documents consist of HTML elements. When writing HTML, we tell browsers how to render elements using HTML tags. Each element consists of an opening tag, a closing tag, and content. For example <h1>I am a heading<h1>. Tags may also have attributes that give further instruction to the browser.

Computers Are literal

Computers interpret instructions literally. This means that any intruction given to a computer needs to be unambiguous. Many bugs arise from ambiguous instructions which have been interpreted with unexpected results.

Inline & Block Elements

HTML elements can be either inline or block. Block elements form an "invisible box" around their content, which can be manipulated to position content.

Lesson 2 - Creating a Structured Document

The "Tree-like" Structure of HTML

The "tree-like " structure refers to the fact that HTML elements can have other elements inside them. This relationship can be drawn as a family tree. In a browser, this structure renders as a series of nested boxes. There are boxes, inside boxes, inside boxes ...

The Relationship Between Indented HTML and Boxes

When you read an HTML document as text (e.g. source code), you see a line of changing indentations on the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is in the tree.

Using Text Editors For Development

Writing HTML/CSS, developers use special text editors like Atom. These editors make development easdier, by automating tasks and providing contextual reference and support.

Lesson 3 - Adding CSS Style and HTML Structure

Avoid Repition - Keep It DRY

It is important to avoid repetition in your code, in order to avoid errors and ensure consistency. When developers repeat themselves, they may end up copying and pasting style attributes into many HTML elements or writing the same code many times. This can lead to errors when the developer decides to make a change to something. If they don't make that same change everywhere the repeated code appears, unexpected problems will arise.

For these reasons, it is good practive to make sure that you Don't Repeat Yourself.

CSS

CSS is an abbreviation of Cascading Style Sheets. CSS gives developers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.

Styles can "cascade" through style sheets (externally referenced files), elements, and classes
